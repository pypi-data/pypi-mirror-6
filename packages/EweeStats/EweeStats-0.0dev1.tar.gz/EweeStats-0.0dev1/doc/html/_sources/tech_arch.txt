Architecture du programme
#########################
Le programme se divise en 3 parties : 
    1. lecture
    2. traitement des données
    3. création du graphique

La lecture et le traitement des données sont dans une même boucle qui continue tant qu'on appuie pas sur le bouton SELECT de l'écran.

La partie création du graphique est effectuée par un autre processus que la boucle principale. Nous verrons ça plus tard. Elle est appelée par la boucle de lecture.
